# In-Memory база данных   



## Функционал
### Сервис реализует базу данных, хранящуюся в оперативной памяти компьютера и реализующую CRUD-операции. Все взаимодействия с БД происходят через терминал. Тип данных - структура UserRecord, которая содержит в себе:

* Уникальный идентификатор пользователя в системе
* Имя и фамилию пользователя
* Возраст пользователя
* Номер телефона пользователя



## Запуск и взаимодействие
### Для запуска необходимо установить [Python](https://www.python.org/downloads/) и склонировать репозиторий данного проекта к себе на пк `git clone https://gitlab.mai.ru/MiASaenko/task_python_1.git`

### После этого необходимо запустить программу с помощью команды `python -m src.db.endpoints`

### После чего все операции с БД будут доступны через пользовательский интерфейс



## Особенности
### В папке backend я добавил отдельную папку с функциями, проверяющими корректность введенных данных(для лучшей читаемости кода), также добавил немного обработки ошибок, чтобы лучше информировать пользователя, что он делает что-то не так и все точки входа в программу объединил в модуль endpoints.


* backend - модуль бизнес-логики
    * errors.py - реализация ошибок
    * memory.py - непосредственно операции БД
* endpoints - слой взаимодействия
    * \_\_main\_\_.py - запуск приложения
    * tui.py - реализация пользовательского интерфейса с отловом ошибок
